## What This Project Is

A local-first YouTube-shorts factory. Feed it a long-form video (podcast, talk, vlog); it transcribes locally, has Claude pick the most clip-worthy story spans, edits each into a tight 9:16 short (filler cut, pace tightened, punch-in reframe, B-roll cutaways, karaoke captions, title card, branding, music), and saves finished `.mp4`s under `./output/<source-slug>/<title-slug>.mp4`. The channel brand is **@C0BALT_CUT** (see `brand/BRAND.md`).

## Architecture

Atomic Claude Code skills, one per video-editing operation. Each skill lives at `.claude/skills/<name>/SKILL.md` and is independently invocable. Skills chain into the full shorts pipeline but are NOT a monolithic codebase.

**Stack:** whisper.cpp (local transcription) + Claude (semantic decisions, via host session or `/crew` tmux members — no API key) + ffmpeg (all media ops) + MediaPipe/OpenCV (faces, saliency) + PIL (all text rendering — the local ffmpeg has NO libass/drawtext, so every text overlay is a transparent PNG composited by ffmpeg).

**No Anthropic API key.** Claude-driven skills (`segment-topics`, `pick-segments`, `verify-coherence`, `bookend-trim`, `trim-filler`, `verify-bookends`, `chunk-captions`, `broll-pick`, `sfx-beats` comedy mode, `generate-title`, `pick-mood`) run via `claude -p` or long-lived Claude tmux panes on the user's subscription. mcptube's own `discover` command needs an LLM API key and is therefore FORBIDDEN — use the mcptube-bundled `yt-dlp` ytsearch instead.

## Entrypoints

| Entrypoint | What it is | When to use |
|---|---|---|
| `start.sh` / `/start` | **Primary orchestrator.** Runs the pipeline across long-lived tmux panes (`shorts-<id>-srcprep`, `-analysis`, and `SHORTS_MAX_PAR` span lanes `-lane-NN`). Each lane owns ONE pooled Claude pane reused across spans/phases with `/clear` between dispatches, so claude processes stay O(max_par), not O(spans). Resumable: each phase skips when its output artifacts exist. | Any "run the pipeline" request |
| `shorts.sh` | Legacy sequential fallback: same skill chain, single process, `claude -p` per step. | Debugging a single skill in isolation, or when tmux is unavailable |

```bash
bash start.sh <youtube-url>      # fresh run
bash start.sh <11-char-yt-id>    # bare ID
bash start.sh work/<source-id>   # reuse an already-ingested source
bash start.sh url1 url2 ...      # batch, sequential per video
bash shorts.sh <url> [n=5] [dmin=20] [dmax=40]
bash .claude/skills/scout-sources/scout-sources.sh   # what to clip NEXT (ranked candidates)
bash .claude/skills/sources-ledger/sources-ledger.sh show   # which sources have been clipped (registry)
bash .claude/skills/reap-source/reap-source.sh --backlog --dry-run   # what disk a cleanup would free
bash .claude/skills/reap-source/reap-source.sh --backlog   # reclaim heavy files from all finished sources
```

`start.sh` phases: **1** srcprep pane (ingest + transcribe) + analysis pane (segment-topics → pick-segments → verify-coherence) → **per-span lanes**: `SHORTS_MAX_PAR` lane workers each pull the next unclaimed span (mkdir-locked counter) and chain its three phases end-to-end — **edit** (bookend-trim → cut/assemble → trim-filler → cut-filler → tighten-pace → verify-bookends → fill-vertical), **captions** (jump-cut → zoom-punch → chunk-captions → switch-faces → broll-pick → broll-composite → fix-cold-open → burn-subtitles → sfx-beats comedy → generate-title → title-transition → brand-overlays → loudnorm), **completion** (like-subscribe-overlay → pick-mood → bg-music → end-card → speed-up → director-pass → qc-clip → visual-cadence → name-short → save-local → grade-clip) — so early spans land in `./output/` while later spans are still editing, and Claude-wait in one lane overlaps ffmpeg in another. broll-cleanup + schedule-drip run once at end (schedule-drip stages the daily drip from the graded clips). Resume markers: edit skips on `clip_NN.vert.mp4` + `.path` sidecars; captions on `clip_NN.leveled.mp4`; completion on `clip_NN.done.completion`. Failures write `clip_NN.fail[.captions|.completion]` and cancel the rest of that span only; every phase attempt deletes its own stale marker first, so retried spans un-skip themselves (shorts-8m6).

## The Pipeline (canonical — every skill below MUST run on a full pipeline invocation)

`scout-sources` is the optional step BEFORE the pipeline: deterministic source discovery (keyless yt-dlp outlier scoring — views/day velocity, views-per-sub ratio, comment engagement, replay-heatmap peakiness, curiosity-hook title bonus) ranking niche-search candidates into `work/_scout/candidates.json`. Feed the winner to `start.sh`. Not part of the per-video chain.

```

  ├─ ingest           (yt-dlp → work/<id>/source.mp4 + ingest.json + heatmap.json [most-replayed graph + chapters; absent on low-view sources, backfilled on cache-hit])
```

**Hard rules for any "rerun the pipeline" / full-pipeline request:**
- Use `fill-vertical` for the 9:16 reframe — punch-in crop that FILLS the frame (face ~45% height, lip-activity speaker pick, saliency crop for no-face shots). NEVER letterbox, NO blur bars. Both `fit-vertical` and `reframe-vertical` are retired/deleted (they linger only in the gitignored stale `.agents/` copy — never resurrect them). It clusters faces across shots to find the dominant speaker, biases the per-shot pick toward whoever is actually talking, and frames a non-speaking reaction/listener shot LOOSER so the short never dwells hero-framed on the wrong person.
- pick-segments builds each short from `cuts` — 1-3 NON-contiguous source ranges within ONE topic, assembled into one story (skip the sag, keep hook→payoff). The `cut-clip` step cuts each range precisely and concats; `assemble.py` joins the clip-local transcript so all downstream skills stay synced. A single-cut short is `cuts:[[t0,t1]]` and takes the plain `rebase.py` path. Multi-cut spans bypass verify-coherence tightening (already tightened by construction).
- pick-segments must choose complete cold-viewer story arcs, not isolated replay highlights. YouTube replay heatmap and RMS are supporting evidence only: use them to discover candidates and break close ties, but never let them outrank setup → turn → landing context. A replay spike often marks the memorable sentence inside a larger explanation; include enough before and after for the clip to make sense and not end abruptly.
- `verify-bookends` runs AFTER `tighten-pace` and BEFORE `fill-vertical`. It is INWARD-ONLY (bookend-trim already had its outward chance) and may drop a span only on cleanliness failures requiring >2s of trim — never on hook weakness alone. Disable with `VERIFY_BOOKENDS=0`. It ALSO runs a COLD-VIEWER CONTEXT GATE (MODIFY 2): classifies whether the FIRST DELIVERED SENTENCE stands alone for a stranger (FAILs on a dependent clause, a pronoun/"it/they/that" with no on-screen referent, a fragment, or flat throat-clearing setup — the "i've got dms on instagram" class) and snaps `t0` FORWARD to the next context-bearing sentence within the `VB_PAYOFF_BUDGET` (default 3.0s) window, under the SAME inward-only/≤2s-removed/≥15s-left guards as the fragment-opener `open_snap` (composing by taking the LARGER justified snap). Cold-context weakness alone is NEVER a drop. It emits `context_pass` + `first_payoff_offset` into `clip_NN.verify.json` for `grade-clip` to consume (reuses the existing head/tail vision call — no new model).
- `verify-completeness` runs AFTER `bookend-trim` and BEFORE `cut-clip`, in SOURCE coordinates. It is the OUTWARD counterpart to verify-bookends: it reads each span's assembled arc (the words inside its `cuts`) plus a tail lookahead and may nudge `t1`/the last cut's end outward to the landing sentence (capped at dmax) when the payoff got cut off. It NEVER drops a span and NEVER extends past dmax. It runs pre-cut because the clip-local transcript discards source timestamps after cut/trim/tighten (rebase.py), so outward extension is only clean in source coords. Non-fatal (passthrough on any failure), idempotent, `VERIFY_COMPLETENESS=0` disables.
- `chunk-captions` runs BEFORE `broll-pick` (b-roll windows snap to whole caption-chunk boundaries — no mid-word cuts).
- `broll-composite` runs AFTER `broll-pick` and BEFORE `burn-subtitles` — captions must burn OVER the cutaways, never under them.
- B-roll cutaways are full-frame hard cuts (entire 1080×1920 replaced, scale-cover + SALIENCY crop toward the action — not blind center — no bars, no crossfade/zoom). Podcast audio is CONTINUOUS — never replaced; no transition SFX by default (pure stream-copy), `BROLL_SFX=1` amix's a short synthesized whoosh onto each cutaway in/out. B-roll audio is always dropped. Bottom-bar/letterbox b-roll = regression.
- `broll-pick` anchors are CONTEXTUAL/scene-level, not literal keyword objects — footage must match the story's tone (a tense "red dot" beat wants a sniper/laser sight, NOT a cat laser toy). Vision verify is given the spoken context and rejects literal-but-wrong matches. Aim dense (~6-10 windows where a sensible visual exists). B-roll files are namespaced per clip (`<clip>_broll_NN.<ext>`) in the shared `broll/` dir — NEVER reuse bare `broll_NN.mp4` slot names across spans (cross-span contamination bug).
- `broll-pick` discovery uses the mcptube-bundled `yt-dlp` ytsearch, NOT `mcptube discover` (the latter needs an LLM API key the stack forbids). Vision verify via Claude; candidate prep (ingest + frame grids) runs CONCURRENTLY and verification is BATCHED — up to `BROLL_BATCH` (default 4) candidates judged per vision round-trip, with `BROLL_VISION_CAP` (default 16) bounding total candidates judged, not calls. Each window gets at most 2 query attempts (original + one literal↔metaphorical rewrite; rewrites are also batched into one call).
- `broll-cleanup` runs exactly ONCE at end of run, evicting only `video_id`s in each `broll_plan.json`'s `ingested_video_ids` — never the podcast source. It must never modify or delete `broll_plan.json`.
- `title-transition` is mandatory and runs AFTER `burn-subtitles`, BEFORE the brand overlays. It is a COLD OPEN, NOT a card: the styled title animates in the TOP banner zone (anchor `TITLE_ANCHOR_FRAC`, default 0.135) OVER the live opening footage so frame 1 is content (a face mid-sentence), never a blocking title card — this is the swipe-rate / "Stayed to Watch" fix. EVERY short uses the SAME single animation — the `glitch` style (RGB-split banner + scanline tear) — there is no per-span style pick. It carries its matched SFX (zaps + crackle) — `styles.py` emits the `events.json` cue list and `sfx.py` synthesizes it, mixed UNDER the live audio so the title's sound matches its animation (`TITLE_SFX=0` disables). NO full-frame bg treatment (the old flash/shake/dim shook the live shot). The title holds until `TITLE_SWAP` (default 2.5s; 2.5s pre-speed → ~2.0s on screen after the 1.25x `speed-up`) and clears via its own fade-out, owning the top banner uncontested at the open; `brand-overlays` (and the standalone `source-credit` fallback) place the source citation in that same top slot only in the FINAL `CREDIT_TAIL` seconds (default 3.0s) so it lands on the closing beat rather than time-sharing the open. The title text comes from `generate-title`. The four other styles (slam/typewriter/bounce/cinematic) and the `pick-title-styles` selection step were REMOVED — glitch is the channel default for all shorts; do not resurrect the others or a style-picking step.
- `bookend-trim` runs AFTER `verify-coherence` and BEFORE `cut-clip`. It snaps each span's `[t0, t1]` to a clean sentence boundary so shorts don't end mid-sentence (whisper output has punctuation stripped, so Claude infers boundaries — heuristics on `.!?` won't work).
- `like-subscribe-overlay` runs AFTER `loudnorm` and BEFORE `bg-music`. It overlays a branded animated CTA (channel gem avatar + @C0BALT_CUT handle + subscribe/like/bell choreography) for ~4s WITHIN THE FIRST THIRD of the clip — start clamped so the CTA ends by the 1/3 mark, floored after the ~2.5s title card — so viewers who drop off early never miss it. `like-subscribe-overlay.sh <in> <out> [dur=4.0] [pos=0.15]`.
- `speed-up` is the LAST edit step — runs AFTER `end-card` and BEFORE `qc-clip` so the QC gate, `visual-cadence`, and `save-local` all act on the delivered clip. It globally retimes the finished short by `SPEED` (default 1.25x) — video `setpts=PTS/SPEED` + pitch-corrected audio `atempo` — a single uniform retime applied last, so ALL upstream relative timing (captions, zoom punches, b-roll windows, CTA, end-card) compresses together and stays in sync; nothing upstream needs to know about it. The one coupling: the cold-open title hold is set pre-speed, so `TITLE_SWAP` defaults to 2.5s → ~2.0s on screen after 1.25x. Deterministic (no Claude), idempotent (`.spmeta`), non-fatal (failure → passthrough). `SPEED=1` or `SPEED_UP=0` disables.
- `end-card` runs AFTER `bg-music` and BEFORE `qc-clip` (and BEFORE `speed-up`). It fades a closing CTA banner ("FOLLOW FOR MORE" + @C0BALT_CUT, Platinum/Sapphire Impact) over the LAST ~2.5s of the finished short so it lands on an intentional beat instead of dead-stopping on a dangling word — the "ends abruptly / no loop or CTA" retention fix. It is TIMELINE-PRESERVING (overlay only; audio stream-copied, duration identical) and centered at `END_CARD_Y_FRAC` (default 0.60), above the caption band. Deterministic, non-fatal (passthrough on failure), `END_CARD=0` skips.
- `jump-cut` cut count is COVERAGE-driven, not a fixed cap: it places enough ~3.2s cuts to churn the WHOLE body so a long clip never trails off into a static tail (the old `JUMP_CUT_MAX=8` left an ~18.8s uncut tail on a 47s clip). `JUMP_CUT_MAX`, when set, is an explicit hard ceiling (back-compat); unset = cover the clip; `JUMP_CUT_MAX_GAP` (default 5s) bounds the step. `visual-cadence` (after `qc-clip`, NON-FATAL) measures the union of all visual changes on the final render via scene-detect and WARNs when the longest static stretch exceeds `MAX_STATIC_GAP` — the keystone signal for the "12 seconds without a pattern interrupt" leak; it never blocks a save (the fix lives in jump-cut coverage).
- `grade-clip` runs at the END of the per-span chain (after `save-local`, grading the work-dir final clip with its co-located sidecars, then mirroring the `grade.json` into `output/<slug>/`) AND standalone (`grade-clip.sh --backlog [output_dir]` → per-clip `<clip>.grade.json` + an aggregate `output/_triage.json`). It is the first skill to inspect a DELIVERED pixel-level artifact for upload-readiness — the on-disk proxy for YouTube's un-exportable VVSA swipe gate, distinct from `qc-clip` (duration/size only) and `visual-cadence` (one diagnostic). It emits `{grade:0-99, tier:GOLD|FIXABLE|DROSS, hard_caps[], signals{...}, fix_routes[]}` per `SELECTION-SUITE-CONTRACT.md`: a deterministic retention-proxy floor (frame1_is_face via MediaPipe+fillplan shot0, letterbox_bars, credit_lit_at_open, first_visual_change, first_payoff_offset, longest_static_gap, opening_caption_words, max_residual_silence, terminal_loop_score) where ANY hard regression (letterbox/face_withheld/credit_at_open/blocking_card/dead_tail) caps `grade≤40`, plus ONE batched Claude rubric (hook↔payoff/open-loop/cold-context) used only when `GRADE_SKIP_CLAUDE=0` (in-chain + backlog default to proxy-only). NON-FATAL (any error → DROSS, exit 0), idempotent (`.gcmeta`), `GRADE_CLIP=0` disables the in-chain grade. `fix-cold-open` and `schedule-drip` consume its `grade.json`.
- `fix-cold-open` is the deterministic, `grade.json`-routed cold-open REPAIR — the on-disk fix for the front swipe-gate leak (cold-open b-roll / mis-crop burying the speaker's face past the 2s gate). It runs gated ops from `fix_routes`: `broll_open_truncate` (drop any `broll_plan.picks` window overlapping `[0, FIXCO_OPEN_GUARD_SEC=2.2]` and re-composite onto the clean pre-b-roll vertical so frame 1 is the speaker), `shot0_repunch` (re-run `fill-vertical` face detection on the 16:9 source when `.vert.fillplan.json` `shots[0].kind != "face"` — re-detects, a real per-clip cost), `credit_rerender`/`card_rerender` (re-fire `brand-overlays`/`title-transition`). PREVENTIVE in-chain (runs AFTER `broll-composite`, BEFORE `burn-subtitles`, on `clip_NN.broll.mp4` with all sidecars + the 16:9 source co-located → all ops clean, emits `clip_NN.fixed.mp4`) vs CURATIVE standalone over a finished `output/<src>/x.mp4` (the defect is baked into pixels with NO recoverable source → it emits `rerun_recommended` and NEVER fabricates a degraded re-crop; letterbox is always `rerun_recommended`). NON-FATAL (any error leaves the input untouched, exit 0), idempotent (`.fixmeta`), `FIX_COLD_OPEN=0` skips the in-chain pass.
- `director-pass` is the AGENTIC, vision-driven QA/repair pass — the open-ended per-clip quality loop layered ON TOP of the deterministic grade→route suite (`grade-clip` is the cheap proxy floor; `fix-cold-open` repairs only its closed cold-open routes). A Claude "director" WATCHES the delivered short (`frames.py` → ONE labelled contact sheet of `DIRECTOR_FRAMES`=12 frames sampled across the WHOLE clip + the transcript scaled into finished-clip time + the fill/broll/cadence/title/mood/grade sidecars) and judges it open-endedly in natural language — catching the long tail of "this feels off" issues ANYWHERE (cold open, dead/rambling tail, tonally-wrong b-roll, a wrong-person punch-in, mistimed captions, a too-hot music bed, a flat hook), not just the cold open and not just a fixed route vocabulary. It binds that judgment to a BOUNDED, pixel-safe applier so it can never corrupt a clip: `tail_trim {t1}` (re-`cut-clip` to `[0,t1]` + re-fire `end-card` so the new tail lands on the CTA beat — PIXEL-SAFE in both modes) and `music_down {volume}` (re-run `bg-music` on the pre-mix `.ctaed.mp4` at a lower bed vol → `end-card` → `speed-up`; preventive/in-chain only). Everything else is SURFACED (recorded with `where` + `rerun_recommended`, NEVER auto-faked) — cold-open/b-roll/punch/caption repairs live upstream of the whole finishing chain, so re-deriving them from a finished clip is the preventive `fix-cold-open`'s job earlier or a pipeline re-run, not a degraded post-hoc re-crop. Runs in `start.sh` phase 4 AFTER `speed-up`, BEFORE `qc-clip`, on `clip_NN.sped.mp4` with all `clip_NN.*` sidecars co-located (preventive mode); standalone `director-pass.sh --backlog` reviews finished `output/<src>/*.mp4` (curative — `tail_trim` only, rest surfaced) → `output/_director.json`. Emits `clip_NN.sped.director.json` (the edit-list report) + `clip_NN.sped.dir.mp4` (only when a fix changed pixels). Re-reviews up to `DIRECTOR_MAX_ITERS` (default 1). NON-FATAL (missing input/sidecars, unreadable clip, failed sub-skill, malformed/empty review → input untouched, `ship`/`disabled` report, exit 0), idempotent (`.dpmeta`), `DIRECTOR_PASS=0` disables. Owns only its dir — re-fires `cut-clip`/`end-card`/`bg-music`/`speed-up` by their published `.sh` contracts.
- `schedule-drip` runs ONCE at end of run (after `broll-cleanup`, after every `grade-clip` verdict) AND standalone — a deterministic greedy scheduler over every `output/**/*.grade.json` that STAGES (never uploads — no API key) a daily drip into `output/_toupload/<date>/` (clip copy + `metadata.txt` = lowercased title + winning-topic hashtags) + `output/_toupload/schedule.json` with `gap_warnings`. It is the only skill that directly attacks the #1 unconfounded leak (the dark gap) and the 53-from-one-source feed-fatigue: GOLD-first gate, rank by grade, gate by the checked-in `topics.scorelist` (GO clips fill days first, HOLD only backfills an otherwise-dark day; HOLD is a denylist that wins over an incidental GO substring), de-dupe near-identical clips (e.g. `-1.25x` speed twins — keep the higher grade), `MAX_PER_SOURCE_PER_DAY=1` round-robin, drop anything in `output/upload-log.json` (absent = nothing posted), and WARN before any dark day. Knobs `POSTS_PER_DAY` (1) / `MAX_PER_SOURCE_PER_DAY` (1) / `DRIP_HORIZON_DAYS` (14); reuses `GRADE_MIN_UPLOAD` (60). NON-FATAL, idempotent (`_toupload` rebuilt each run), `SCHEDULE_DRIP=0` skips. The full immediate-payoff sequence is `grade-clip --backlog` → `output/_triage.json` → `fix-cold-open` on the FIXABLE set → `schedule-drip`.
- **Disk hygiene (`sources-ledger` + `reap-source`).** `work/` accumulates ~25× the size of the finished shorts: the full podcast `source.mp4` (re-downloadable) and every per-stage `clip_NN.*.mp4` intermediate linger forever while the deliverable is already safe in `output/`. Two atomic skills manage this, split by the user's chosen policy (manual cleanup, delete the re-downloadable source, both a registry file and a bd memory). `sources-ledger` (runs ONCE at end of every run + standalone `sync`) maintains `work/sources.json` — the registry of which sources have been clipped (title, url, duration, produced shorts + grades, disk footprint, `active|reaped` status) — plus a keyed `bd remember` per source so future sessions recall it and can re-ingest from the saved URL. `reap-source` is the MANUAL cleanup tool (never auto-runs): `reap-source.sh <id>` (or `--backlog`, or `--dry-run`) deletes the heavy reclaimable artifacts (`source.mp4`, `clip_*.{mp4,mov,wav,m4a,webm,mkv}`, the `broll/` cache, and the source's mcptube ingest via `broll-cleanup`) and KEEPS all lightweight JSON (`ingest`/`transcript`/`topics`/`segments`/`grade`/`broll_plan`) as the on-disk memory, writes a `.reaped` marker, then re-records the ledger. It is GATED — refuses a source with no finished `output/<slug>/*.mp4` unless `--force` — so a half-run source is never reaped before it delivers. Restore a reaped source with `bash start.sh work/<id>` (re-ingests from the URL; `transcript.json` is kept so re-transcription is skipped). Idempotent, non-fatal.
- **RLM-assisted `segment-topics` (long sources).** Chunking is SEAM-AWARE: `build_rlm_prompt.py` cuts windows on natural seams (long silences = large inter-segment gaps; topic-shift cue phrases) near the `RLM_TOPICS_CHUNK_SEC` target with `RLM_SEAM_OVERLAP` (45s) overlap — never blind fixed windows. Per-chunk MAP runs on **Sonnet** via the PROJECT-LOCAL `rlm-segment-subcall` agent (the candidate "most clip-worthy moment" judgment is the quality call Haiku under-rates; the global `rlm-subcall` stays Haiku for `/rlm`); `RLM_SUBCALL_MODEL` overrides. The orchestrator coverage-VERIFIES each chunk's topics tile its window and re-dispatches gappy ones before REDUCE (`parse_reply.py` warns + force-repairs as the safety net). Candidates carry a `confidence` (0–1) so `candidates.hint.json` is RANKED, not flat, and `pick-segments` surfaces ranked hints. MAP results CACHE under `work/<id>/rlm/map/chunk_NN.<hash>.json` keyed by content hash (a re-run embeds unchanged chunks, dispatches only changed ones; stale-hash files pruned); `work/<id>/rlm/usage.json` logs per-chunk seconds/words/≈input-tokens/cached/model for tuning (true leaf token counts aren't exposed by the Task-subagent path, so it's a chars/4 proxy + structure). `RLM_THREADS=0` disables cross-chunk threading; `RLM_CACHE=0` disables the cache. Full ledger of adopted + explicitly-NOT-adopted (true recursion depth>1, learned activation gating) tactics: `.claude/skills/segment-topics/RLM.md`.
- **Cross-chunk THREAD spans.** A narrative thread that opens early and pays off far later (proven: JRE #2217 Brian Cox info-paradox, setup @524s ↔ payoff @9740s) can become ONE short. Detection is in the RLM REDUCE: it clusters all candidates + map-emitted `open_threads`/`callbacks` into setup→payoff/callback/escalation/contradiction threads and emits COMPOUND candidates (`thread:true`, `kind`, ordered `cuts` ≤3, `bridge`, `confidence`); `pick-segments` surfaces them in a CROSS-CHUNK THREADS block. A pick marked `thread:true` with ≥2 cuts is the ONLY span allowed to cross topic boundaries — it bypasses the single-topic drop in `pick-segments/parse_reply.py` and (being multi-cut) already bypasses `verify-coherence` tightening; `assemble.py` builds it range-agnostically. Guardrails (verify-completeness, verify-bookends, director-pass) catch a bad stitch. Capped at 3 cuts, only on a genuine thread.
- `selection-report` writes `output/<slug>/_selection.json` for each source (ONCE at end of run after `sources-ledger` + standalone `--backlog`): the SHIPPED shorts (scores + rationale from `segments.raw.json`, linked to their delivered filename via `clip_NN.done.completion`) ALONGSIDE the CONSIDERED-not-shipped rlm candidate menu (each marked picked/unused by overlapping its `[t0,t1]` vs the shipped cuts) + the topic list. Deterministic, no Claude, idempotent, non-fatal — the audit artifact that shows the arguments the pipeline passed over next to the ones it shipped.
- Mid-sentence opener guard: `pick-segments` drops spans whose first word is a sentence-fragment opener (`of/than/nor/whom/whose/...`, high-precision so temporal hooks like "In 2008..." survive), and `verify-bookends` deterministically snaps `t0` FORWARD off a fragment opener (inward-only, ≤2s removed, ≥15s left) even when the vision pass returns keep — so a short can never open mid-clause ("of picturing what's happened...").
- `brand-overlays` runs AFTER `title-transition` and BEFORE `loudnorm`. It bakes BOTH persistent brand marks in one encode: the "Original video: <title>" credit as a TOP chyron (banner top at y≈4%, title read from `work/<id>/ingest.json`, clear of the lower-third captions and the centered title card) and the `@C0BALT_CUT` channel mark bottom-center (bottom-anchored at y≈97.5% — the vertical opposite). Brand colors from `brand/BRAND.md`: Platinum `#E8ECF1` type with the slashed-zero in Sapphire Glow `#2E6BFF`. The CTA overlay composites on top of the watermark for ~4s within the first third, which is intended. The standalone `source-credit` and `watermark` skills remain the single-overlay atomic ops (and the orchestrator's fallback path) and own the PNG renderers brand-overlays reuses.
- The caption/accent blue everywhere (burn-subtitles active word, title-transition accent word, source-credit label, watermark zero, CTA accent) is Sapphire Glow `#2E6BFF` from `brand/BRAND.md` — matched to the channel pfp/banner gem. Electric cyan `#00E5FF` is retired; reintroducing it is a regression. All overlay text is Impact (`/System/Library/Fonts/Supplemental/Impact.ttf`) with a thick black stroke.
- Final shorts are named from the title: `name-short` slugs `generate-title`'s output into `<kebab-title>.mp4`, and `save-local` puts it in `output/<source-title-slug>/`. Generic `short_NN.mp4` names in output/ = the orchestrator forgot to pass the name through.
- If `start.sh`/`shorts.sh` does not invoke every skill above in the listed order, the entrypoint is wrong — fix the entrypoint, do not silently skip skills.
- Verify after a run: every saved `output/<source-slug>/*.mp4` must be 1080x1920 (full-bleed punch-in, NO blur bars), open with a COLD-OPEN top-banner title over live footage for ~2s (NOT a blocking centered card; the title carries its own style-matched SFX), show the source citation in the top banner over the FINAL ~3s, AND have a CTA card for ~4s landing within the first third of the clip. If any is missing, the pipeline regressed.
- `zoom-punch` runs AFTER `fill-vertical` and BEFORE `broll-pick`/`broll-composite`/`burn-subtitles` — cutaways replace the full frame and captions must not wobble, so the punch zooms only ever touch the clean vertical. Deterministic (no Claude), non-fatal (failure → unzoomed passthrough), `ZOOM_PUNCH=0` disables.
- `switch-faces` runs AFTER `zoom-punch`/`chunk-captions` and BEFORE `broll-pick` (on the clean vertical, before cutaways/captions). It manufactures cut-to-listener reaction shots: at phrase boundaries (chunk-caption ends — the semantic pause, since `tighten-pace` has already collapsed every raw audio gap to ~0.08-0.15s) it hard-cuts to a NON-speaking second face cropped from the 16:9 source for ~0.8-1.2s, then back. It reuses `fill-vertical`'s face detection + identity clustering, and fires ONLY on a TRUE two-shot — a frame where the dominant speaker AND a non-dom face co-occur — so single-interview/talking-head footage (where identity drift would otherwise spawn spurious "second" identities) passes through untouched. CRITICAL: TIMELINE-PRESERVING — it overlays the listener crop onto the base via `enable=between(t,…)`, audio is stream-copied and duration is identical, so every downstream timestamp stays valid. Deterministic (no Claude), non-fatal (failure / no two-shot / no chunks → passthrough), `SWITCH_FACES=0` disables. The cut-to-listener counterpart to `jump-cut`'s same-speaker reframes and `zoom-punch`'s same-speaker punches.
- `jump-cut` runs AFTER `fill-vertical` and BEFORE `zoom-punch` (both touch only the clean vertical, before cutaways and captions). It manufactures "multi-cam" visual churn on static talking-head stretches: it tiles the clip into ~3.2s segments at word-start boundaries and hard-cuts between a base framing and tighter ~1.1-1.16x punch-in reframings of the SAME speaker (the cold-open lead + the tail stay base). It is the visual-change-density lever for sources where b-roll stays sparse — b-roll cutaways later override their own windows and captions burn on top, so neither warps. CRITICAL: it is TIMELINE-PRESERVING — audio is copied untouched and total duration is identical, so every downstream timestamp (captions, b-roll windows) stays valid; it adds cuts, never trims. Deterministic (no Claude), non-fatal (failure → passthrough), `JUMP_CUT=0` disables.
- `sfx-beats` comedy mode IS canonical: it runs AFTER `burn-subtitles` and BEFORE `generate-title`/`title-transition`. Claude marks 0-4 punchline/irony/insight beats (vine boom / record scratch / ding); marking ZERO beats on a non-comedic clip is correct behavior and passes through. Non-fatal, `SFX_COMEDY=0` disables. The tension mode (riser/hit/stinger) stays NOT canonical — only on request.
- `SHORTS_DMIN`/`SHORTS_DMAX` default to 28/55 (June 14 re-tune from 20/40, which had over-truncated). These are the SOURCE-SELECTION budget, NOT the delivered runtime: pick-segments selects generously within them and downstream trim-filler/tighten-pace/verify-bookends shave ~20-30%, landing the DELIVERED short in the ~30-40s sweet spot. The 40→55 history: June 12's Growth suite dropped 60→40, but combined with the downstream cutters that delivered ~26-32s ("truncated" feeling); 55 restores complete-feeling arcs without resurrecting the dead 60s back-half. Override per-run when a story truly needs more room.

## Claude Dispatch (pane.sh)

`_lib/pane.sh` provides `run_claude_step <step> <prompt> <reply>`: with no `SHORTS_PANE` it runs `claude -p --output-format text`; with `SHORTS_PANE=<tmux-session>` + `SHORTS_PANE_MODE=chat` (what start.sh uses) it messages a long-lived interactive Claude pane, which writes `out.txt` + touches `out.done`, polled every `PANE_TICK` (6s) up to `PANE_TIMEOUT` (1800s). Claude-driven skills all follow the same shape: `build_prompt.py` → `run_claude_step` → `parse_reply.py` (with a deterministic fallback if the reply doesn't parse).

## Conventions

- **One skill per atomic op.** Never bundle two operations into one skill. Tempting helpers (e.g. "transcribe + burn-subtitles") belong as separate skills that share I/O contracts.
- **JSON between skills.** Inter-skill data passes as JSON files on disk (transcript.json, topics.json, segments.json, etc.). Reading and writing is cheaper than threading state, and any skill can be re-run independently.
- **Every skill is idempotent.** mtime + `.*meta` signature checks mean re-running a skill with unchanged inputs is a no-op — safe to re-invoke any stage.
- **Source-of-truth paths in `.env`.** Binary paths (whisper-cli, model file) live in `.env`; never hardcode.
- **No CLAUDE.md per skill.** A single SKILL.md with frontmatter is the contract.
- **Claude-skill shape.** `build_prompt.py` → `run_claude_step` (pane.sh) → `parse_reply.py` with a deterministic fallback. Batch all spans into ONE Claude call where possible.
